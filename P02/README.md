# Projecte de Gestió amb GitHub

Aquest repositori conté totes les activitats i productes del nostre projecte, organitzats de manera clara i uniforme per facilitar la gestió i el lliurament.  

## Estructura del Repositori

Per una correcta organització, cada activitat i producte té la seva pròpia carpeta amb la documentació i els arxius necessaris:

- **tasca01/**  
  Conté l’enunciat i la solució de la primera tasca.  
  [Llegir README de tasca01](tasca01/README.md)

- **tasca02/**  
  Conté l’enunciat i la solució de la segona tasca.  
  [Llegir README de tasca02](tasca02/README.md)

- **tasca03/**  
  Conté l’enunciat i la solució de la tercera tasca.  
  [Llegir README de tasca03](tasca03/README.md)

*(Afegir més carpetes de tasques segons sigui necessari)*

- **producte04/**  
  Conté el producte final amb tot el material lliurable corresponent.  
  [Llegir README de producte04](producte04/README.md)

## Guia de Bones Pràctiques

- Cada carpeta té un **README.md** amb l’enunciat, la descripció i l’enllaç al fitxer amb la solució.  
- Mantingueu el mateix criteri d’organització per a totes les activitats i productes.  
- Actualitzeu els arxius de manera regular per assegurar que la documentació està completa i correcta.

## Objectiu

Aquest esquema facilita:

- Una visió clara de totes les tasques i productes del projecte.  
- El seguiment del treball realitzat i pendents.  
- La correcta presentació i lliurament del projecte al final del curs.
