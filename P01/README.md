# Projecte amb Metodologia Kanban

## Descripció del projecte
Aquest projecte segueix la metodologia **Kanban**, igual que en projectes anteriors. L’objectiu principal és assegurar un bon seguiment de les tasques i del desenvolupament del projecte, garantint que tots els membres de l’equip estiguin coordinats i informats de l’estat de cada activitat.

## Product Backlog
El projecte parteix d’un **product backlog** proporcionat, que conté totes les tasques i funcionalitats que cal desenvolupar. Aquest backlog és la base per planificar, organitzar i prioritzar el treball.

## Seguiment amb Kanban
- Cada tasca del backlog s’ha de traslladar al **Kanban Board** (Planner) per fer-ne el seguiment.
- El Kanban Board s’utilitza per visualitzar l’estat de les tasques: **Per fer**, **En procés**, **Finalitzat**.
- És imprescindible actualitzar les tasques regularment per reflectir el seu estat actual.

## Revisió Setmanal
- Cada setmana es farà una revisió de l’estat del **Planner**.
- Cal mantenir el tauler actualitzat de manera contínua; **no deixar les actualitzacions per l’últim moment**.
- La revisió servirà per identificar possibles bloquejos i reajustar prioritats si és necessari.

## Bon Pràctica
- Assignar responsables a cada tasca.
- Establir dates límit raonables.
- Documentar els canvis i avanços al Kanban Board per tenir un historial clar del projecte.

## Conclusió
Seguint aquest mètode, el projecte es desenvoluparà de manera ordenada i transparent, facilitant la col·laboració i assegurant que totes les tasques es completen dins dels terminis establerts.
